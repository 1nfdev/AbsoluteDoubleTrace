 Trace Features
Canvas Fingerprint Protection Fetching Status... Configure
Canvas Fingerprinting Protection
This feature will, simply put: generate a new canvas ID every minute to make it impossible to properly identify you as you browse the web.
Canvas Fingerprinting is a relatively new tracking technology and it has started to be more widely used as time has gone on.

To see this in action visit https://panopticlick.eff.org/ and check the 'Hash of canvas fingerprint', you should see it change every minute.
Audio Fingerprint Protection Fetching Status... Configure
Audio Fingerprinting Protection
This will cause the Audio Context API to not work at all.
So as a result sounds could fail to play on some sites, in most cases it will be fine, but if you are using websites that are for MIDI or have some form of Audio analysis on them then this may cause them to not work.
In some cases this protection may break some sites due to causing errors in code, even ones which aren't related to music/sound.
This feature should only be used by users who know what they're doing.

Check protection at: https://audiofingerprint.openwpm.com
WebRTC Protection Fetching Status... Configure
WebRTC Protection
Protects against WebRTC Local IP Address Leaks
This is important because without it, websites can see your computers local IP address which is often a very good unique identifier for them to use to track you.
Check your protection is working: https://diafygi.github.io/webrtc-ips/
Ping Protection Fetching Status... Configure
Ping protection
This will render the navigator.sendBeacon function unusable and also disable "ping" requests.

This function was specifically designed for tracking, it is used to send a small amount of data, usually when the page is unloading or links have been clicked to a target server. By enabling this, those requests are cancelled before they are sent.
Screen Resolution Tracking Fetching Status... Configure
JS Screen Variable Randomisation
Javscript has an object called "screen" that provides websites with information about your screen resolution.
This can be used to track you because it's a piece of information that always stays constant unless you change your screen resolution in your display settings, which often isn't the best idea.
Enabling this setting will add a random offset between -10 and +10 to most values in the object on each page reload.
This won't stop websites measuring the height and width of the HTML body.

To see this setting in action check the "screen resolution" on BrowserLeaks.com and for more information, see Trace's 'Info' section.
Hardware Fingerprint Protection (New!) Fetching Status... Configure
Hardware Spoofing
There are a number of Javascript variables and APIs that allow websites to know the configuration of your hardware which is more than likely unique, especially when you factor in things such as the GPU.

To protect against this, Trace changes the values of 2 navigator properties - hardwareConcurrency (CPU Core count) and deviceMemory (RAM available to browser).
When enabled, Trace also protects against WebGL fingerprinting - this stops websites being able to see your GPU model.
Battery API Protection Fetching Status...
JS Battery API Protection
This is just another data point that can be used to fingerprint you. It can tell a website about your battery percent, charging state and from that it can be inferred you're using a laptop, phone or tablet. Disabling this API shouldn't affect websites too much.
Network Information API Fetching Status...
Network Information API
navigator.connection returns information about your current internet connection speed,
if you don't want sites knowing that enable this to return the same value every time.
More configuration options coming soon.
 Advanced Features
Referer Controller (New!) Fetching Status... Configure
HTTP "Referer" Header Removal
Stops sending the HTTP 'Referer' header in certain cases; the header itself tells websites the URL that referred you to that page.
From Wikipedia: Referer logging is used to allow websites and web servers to identify where people are visiting them from, for promotional or statistical purposes
Some websites don't like it when browsers don't send the HTTP referer header, so please keep that in mind when enabling this.
Cookie Eater (New!) Fetching Status... Configure
Cookie Eater

Cookies are the main form of tracking used on the web, they store a unique identifier that is used to tie you with everything you do online.

Trace protects you in a unique way, instead of deleting the cookies from your machine, Trace intercepts the network requests and reads the Set-Cookie and Cookie headers, evaluates their contents depending on your settings and then will deal with them appropriately.
Google Header Removal Fetching Status... Configure
Removal of Google Headers
Removing these headers makes you less trackable when using Google services and the Chrome Browser whilst also saving a tiny bit of bandwidth.

Please note:
When testing, I found that the headers weren't always removed, this isn't a problem with Trace, but a problem with the browser not allowing Trace to remove the headers. Unfortunately there is no way around this that I'm currently aware of.
Removing some of these headers (Notably X-Client-Data) can also cause strange behaviour on Google Sites.

Information: https://www.google.co.uk/chrome/browser/privacy/whitepaper.html#variations
User-Agent Randomiser Fetching Status... Configure
User-Agent Randomisation
User-Agent Randomisation protection protects you by changing your HTTP User-Agent every 15 seconds.
A User-Agent is a string of text sent to every website you visit telling it what OS and Browser you're using and their versions.
Whilst this information can be used to show you relevant information for your computer it can also be used to identify you, by changing it very often you make yourself harder to track online.

This feature has been known to break some websites in some cases and on others cause popups about the OS or Browser.
Proxy IP Header Spoofing Fetching Status... Configure
Proxy IP Header Spoofing
IP Spoofing is done by telling the site you're connecting to that you are a proxy doing the request on behalf of another computer. In some cases the site will then take the IP in the headers as your IP, making the IP spoof successful.
This can break some sites and is by no means anywhere near as secure as a VPN, to properly spoof your IP you need a VPN. Please also note that the site you are connecting to will still see your IP as the connecting IP.
getClientRects Protection (New!) Fetching Status...
getClientRects Protection
This Javascript function returns coordinates and size information about HTML elements on a webpage, which seems rather innocent in itself, however since all of the values returned are in pixels.

Drawing the same element(s) on different displays and devices will yield different results due to a number of factors such as the size of the screen, browser viewport and pixel density of the screen, this is done so that the element looks consistent no matter the device.

However tracking companies can and do use this function to create an identifier which is unique to you, which is dangerous because just like Canvas/Audio fingerprinting it is a fairly consistent method of tracking without cookies.

Trace protects you by adding a randomised decimal offset to each value, this will allow you to successfully spoof the fingerprint without breaking websites.
JS Plugin Hide Fetching Status...
JS Plugin Hide
This feature will hide plugins from websites by returning an empty string when navigator.plugins is accessed. This can break some sites and make others look a bit weird, but it's a good privacy enhancement if you enable it.

This doesn't stop the plugins working, just stops a list of them being accessible to webpages.
Please note: You should only enable this if you know what you're doing as it can cause problems on many websites.
E-Tag Tracking Protection Fetching Status...
Removal of HTTP E-Tag Headers
This will stop the E-Tag header from working.
This header has been seen creating respawnable tracking cookies on quite a few websites.
Its main use is for cache but it can be just as easily used as another method to fingerprint your browser as demoed here:
https://lucb1e.com/rp/cookielesscookies/

Disable Network Prediction
Stops chrome allowing sites to perform DNS lookups, SSL/TLS handshakes and TCP connections preemptively.
By setting this to "Protected" you are actively slowing down your browsing experience.. Only by a few milliseconds though.
Alternate Error Pages Fetching Status...
Disable Alternate Error Pages
When unprotected, Chrome uses a web service to help resolve navigation errors and may log information about you.
WebForm Autofill Fetching Status...
Disable Autofill for HTML Forms
Recently advertisers and tracking companies were found to have been hiding hidden form elements in webpages to trick browsers and password managers into autofilling form information which would then be sent off to a server, since this information almost always remains the same, it's a perfect way to track users.

By setting this setting to "Protected" you will lose the convenience of autofill but be protected further against trackers, be sure to also check that you don't have a password manager automatically filling in forms and other text boxes around the web without your knowledge.
Safe Browsing Extended Reporting Fetching Status...
Disable Safe Browsing Extended Reporting
When unprotected, Chrome will send additional information to Google when SafeBrowsing blocks a page, such as the content of the blocked page, your IP address and the time of the request.
Hyperlink Auditing Fetching Status...
Hyperlink Auditing Protection
Hyperlink Auditing is a tool used by websites to log what links you click on a page. Enabling this setting is highly recommended.
This setting can be changed in the "flags" page of your browser but for ease of access it has been added to Trace's settings.

 Web Request Controller
This advanced feature allows you to be in control of the web requests your browser makes. Use it to block tracking domains, hostnames, common javascript tracking scripts and even top level domains. Not to mention the ability to clean URLs to get rid of bad parameters that are used to track you.

 Bad TLD Protection
A Top Level Domain or "TLD" is the classification of domain, e.g. ".com" is a TLD which tells the user the site is "commercial".

In recent years, a lot of new TLDs have popped up. Some are very cheap and easy to buy in bulk, or are not very well policed, and so companies and individuals have taken advantage of them and used them to house Tracking domains or even Malware Domains.

If you are unsure about which TLDs to block and which to allow then there are 4 presets to help you depending on the level of protection you want. It is safe to block all of them and if you choose to do so it likely won't affect your browsing experience, however, if it does you can always disable the individual TLDs.

 URL Tracking Cleaner (New!)
URL query strings (The bits after the question mark) and fragments (The bits after the hash) are often used to track you, the most common ones that you'll see are ones that start with utm_, these parameters are vital for most websites but some of them often contain information about you such as where you came from, the ad you clicked, or some other form of identifier.
Enabling this setting will allow Trace to remove or randomise these parameters so that you are much harder to track.

 Privacy Policy
Last Updated: 2018-05-12 00:54

When reporting URLs, only the URL, current browser (e.g. Chrome/Firefox) and message you write will be sent to the server and recorded. No other information about that transaction is stored.

Error reporting is optional, but when it is turned on, only information about the code will be sent to the AbsoluteDouble servers, this is so errors can be recreated and solved in future updates.

Any and all information is only stored for the length of time it is needed and will be deleted after 1 month at the most. Trace will attempt to make secure HTTPS connections to servers when downloading blocklists and uploading reports. If you have any questions or concerns at all regarding your privacy or Trace in general, please email me. Trace does not and will never collect your browsing data.

Use of Cloudflare
Like many websites, AbsoluteDouble uses Cloudflare, please make sure you agree to their privacy policy before browsing this website. If you do not agree to the privacy policy and still wish to use Trace, you can turn off error reporting and the web request controller to stop all communication with absolutedouble.co.uk.

Regarding Trace Premium
It is possible for us to see your name and PayPal email address when you buy Trace Premium due to how PayPal works and when you buy Trace Premium your code will be stored on the server. Please be sure to read Paypal's Privacy Policy before purchasing Trace Premium.
